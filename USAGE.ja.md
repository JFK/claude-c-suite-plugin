# 使い方ガイド

Claude C-Suite Plugin を日々の開発で活用するための実践ガイドです。

> インストール手順は [README.ja.md](./README.ja.md#インストール) を参照してください。

---

## 60秒クイックスタート

1. 任意の git リポジトリに `cd` する
2. 次を実行：
   ```
   /claude-c-suite:ceo
   ```
3. 推奨事項を読む。CEO コマンドが自動的にプロジェクトを診断し、最も関連の深い3つの経営視点を選び、優先順位付きのアクションリストを返します。

それだけです。**迷ったら `/ceo` から始めてください** — 常に正しい安全な入り口として設計されています。

## 3つのエントリーポイント：`/ask`、`/<role>`、`/ceo`

「どこまで自分で考えたいか」のレベルに応じて3つの入口があります：

| エントリーポイント | 視点数 | コスト | 使いどころ |
|---|---|---|---|
| **`/claude-c-suite:ask <質問>`** | 1（自動選定） | 最安 | 「とにかく答えがほしい・ロール選びたくない」 |
| **`/claude-c-suite:<role> <質問>`** | 1（自分で指定） | 安い | 「これはCTO案件だと分かってる」 |
| **`/claude-c-suite:ceo <質問>`** | 3 + 統合 | 最高 | 「複数視点のトレードオフ解決が要る」 |

`/ask` は最新かつ最もシンプル。質問を11ロール全部に対してスコアリングし、最も鋭い1名を選んでその視点で回答します。質問が本当に複数視点を要するもの（例：「今リリースすべきか先に強化すべきか」）の場合、`/ask` は**ルーティングを辞退**し `/ceo` を案内します。この正直な辞退は機能の一部です — 不適切な単一視点の回答はリダイレクトより有害だからです。

```
/claude-c-suite:ask このSQLスキーマは正規化できてる？
/claude-c-suite:ask 新しく追加した依存関係はどれくらいリスクある？
/claude-c-suite:ask READMEの構造はSEO的に良い？
```

## 2つのモード：質問と レビュー

すべてのコマンドは2つのモードで動作します。プラグインが入力からどちらを使いたいか自動検出します。

### 質問モード

自然言語で質問するだけ。実際のコードベースに基づいて、その役職の視点で回答します。

```
/claude-c-suite:cto モノレポに移行すべき？
/claude-c-suite:cso JWTの実装は安全？
/claude-c-suite:clo このGPLライブラリをSaaSで使える？
/claude-c-suite:cfo データベースのティアはオーバープロビジョニング？
/claude-c-suite:pm このバグを直すためにリリースを遅らせるべき？
```

役職は質問に答えるために必要なコンテキストだけを集めます — 速くて焦点が絞られています。

### レビューモード

スコープキーワード（または何もなし）を渡すと構造化された分析が始まります。

```
/claude-c-suite:cto                 # フルレビュー
/claude-c-suite:cto debt            # 技術的負債に絞る
/claude-c-suite:cso auth            # 認証に絞る
/claude-c-suite:clo licenses        # 依存関係のライセンスに絞る
/claude-c-suite:coo cicd            # CI/CDパイプラインに絞る
/claude-c-suite:cmo seo             # SEOに絞る
/claude-c-suite:caio models         # モデルライフサイクルに絞る
/claude-c-suite:cfo costs           # クラウドコストに絞る
/claude-c-suite:cio data            # データガバナンスに絞る
```

各コマンドの受け付けるスコープはコマンドファイルのフロントマターに記載されています（引数なしで実行するとデフォルトレビューが見られます）。

## よく使うパターン

### 複数ロールでの相談

同じセッションで複数のロールを実行すると、各ロールは既に呼んだ他ロールの分析結果を自動参照します。

**例：リリース前準備の流れ**

```
1. /claude-c-suite:cto       → 技術的負債のブロッカーを特定
2. /claude-c-suite:cso       → 負債の修正が認証ギャップも閉じることに気づく
3. /claude-c-suite:qa-lead   → 認証修正周辺で未テストのコード経路を発見
4. /claude-c-suite:pm        → 上記すべてを踏まえてマイルストーンを再整理
```

`/pm` を実行する時点で、`cto`、`cso`、`qa-lead` の所見がコンテキストにあり、それらを優先度付けに反映します。

### CEO に司令塔をやらせる

どのロールに相談すべきか分からない場合、まず `/ceo` を実行してください。状況に最適な3つの視点を選び、統合された意思決定として返します。その後、必要に応じて個別ロールに掘り下げていけます。

```
/claude-c-suite:ceo ローンチ準備は整っている？
```

出力には選ばれた3名のCxOとその理由が示されるので、次に何を呼ぶべきかの良いガイドになります。

### スコープを絞って時間短縮

大きなリポジトリではフルレビューに時間がかかります。スコープキーワードで絞りましょう。

```
/claude-c-suite:cso auth        # 認証コードのみ、リポ全体ではない
/claude-c-suite:cto debt        # 技術的負債のみ、ドキュメント/バグ/アーキはスキップ
```

## 他のプラグインとの組み合わせ

### [claude-phd-panel](https://github.com/JFK/claude-phd-panel-plugin) と併用

PhD Panel は学術的な深さ（分散システム、統計、データベース理論など）を加えます。先に PhD Panel コマンドを実行すると、その後の C-Suite コマンドはすべて学術的所見を取り込みます。

```
1. /claude-phd-panel:dist-sys   → 合意形成の正しさを学術レビュー
2. /claude-c-suite:cto          → CTOが学術的批評を取り込む
3. /claude-c-suite:caio         → CAIOがdist-sysの懸念をAI展開に反映
```

### [expert-craft](https://github.com/JFK/expert-craft-plugin) と併用

カスタムドメインエキスパートを作成し、C-Suite の所見を相互参照させます：

```
1. /expert-craft cryptographer  → カスタム暗号エキスパートを作成
2. /claude-c-suite:cso          → 標準的なセキュリティレビュー
3. /cryptographer               → CSOの所見を参照しながら深い暗号レビュー
```

## このプラグインがやらないこと

- **コード変更はしない** — すべてのコマンドは分析と推奨のみを返します。何を実行するかはあなたが決めます。
- **権威ある法務/セキュリティ/財務助言ではない** — 出力はAI生成です。重要な推奨事項は、行動する前に専門家に確認してください。すべてのコマンド出力の末尾にディスクレーマーがあります。
- **テレメトリーなし** — プラグインは外部にデータを送信しません。あなたの既存の認証情報で `gh` と `git` コマンドを呼ぶだけです。

## トラブルシューティング

### 「質問したのにレビューモードで答えられた」（または逆）

プラグインは入力に疑問詞や `?` が含まれているかで質問/レビューを自動検出します。検出を間違えた場合：

- **強制的にレビューモードへ**: スコープキーワードを渡す
  ```
  /claude-c-suite:cto full
  ```
- **強制的に質問モードへ**: 入力の末尾に `?` を付ける
  ```
  /claude-c-suite:cto main ブランチを再レビューする？
  ```

### 「`--repo owner/repo` を使おうとして失敗した」

古いバージョンを使っています。プラグインは `owner/repo` 引数を受け付けなくなりました — 現在のワーキングディレクトリのリポジトリで動作します。対象のリポジトリに `cd` するか、v1.4 以降にアップグレードしてください。

### 「存在しないファイルを参照している」

LLM のハルシネーションは特に大規模なコードベースで起こり得ます。行動する前に必ず参照を検証してください。頻繁にハルシネーションが出る場合は、コマンド名・入力・実行したモデルを添えて Issue を報告してください。

### 「13ロール以外のカスタムロールが欲しい」

v1.4 / v1.5 のフィーチャーフリーズ期間中は、新ロールの追加はありません。[expert-craft](https://github.com/JFK/expert-craft-plugin) を使って自分で作ってください — C-Suite とシームレスに統合されます。

## メンテナ向け: audit コマンド

このプラグイン（またはそのフォーク）をメンテナンスする場合、audit コマンドをリリースゲートとして使用してください：

```
/claude-c-suite:audit              # 監査を実行し、合格/失敗を報告
/claude-c-suite:audit matrix       # 整合性マトリクスも併せて表示
```

監査に合格すると、リリース手順が示されます。失敗した場合は、各失敗について修正案が提示され、修正後に再実行するよう促されます。audit コマンドは読み取り専用で、ファイルを一切変更しません。

ターミナルから直接スクリプトを実行することもできます：

```bash
python3 scripts/audit.py
python3 scripts/audit.py --matrix
```

すべてのチェックの一覧は [AUDIT.md](./AUDIT.md) を参照してください。
